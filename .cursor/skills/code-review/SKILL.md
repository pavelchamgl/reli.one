# Code Review Skill

Use this skill when reviewing changed code.

Review checklist:
1. Read full changed files, not only diff.
2. Check project rules.
3. Check API compatibility.
4. Check database/migration risk.
5. Check security risk.
6. Check tests.
7. Check duplicated logic.
8. Check error handling.
9. Check transactions for multi-model writes.
10. Check idempotency for webhooks.

Output:
- Approved / Needs changes
- Critical issues
- Warnings
- Suggestions
- Missing tests
