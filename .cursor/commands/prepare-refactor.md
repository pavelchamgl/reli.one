Prepare a safe refactor plan.

Do not modify code.

For the selected file/domain:
1. Describe current behavior.
2. Identify public API contracts.
3. Identify models involved.
4. Identify tests that exist or are missing.
5. Identify risks.
6. Propose a step-by-step refactor plan.
7. Split the refactor into small commits.
8. Recommend tests to add before changing code.
