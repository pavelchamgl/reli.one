# Task 026 — Политика нормализации концов строк (EOL) через .gitattributes

**Priority:** P3
**Complexity:** Low
**Status:** Done (repo-scope, forward-only)

## Цель

Устранить «фантомные» diff'ы, когда git показывает файл как полностью изменённый
из‑за смены концов строк (CRLF ↔ LF) при идентичном содержимом. Зафиксировать
единый стандарт окончаний строк (LF) для текстовых файлов репозитория через
`.gitattributes`, чтобы такие ложные изменения не возникали впредь.

## Контекст

- Триггер: `Frontend/Frontend3/src/utils/sellerCatalogI18n.js` показывался в
  `git status` как изменённый, хотя содержимое идентично. Причина —
  концы строк: в индексе файл хранится с **CRLF**, в рабочей копии стал **LF**
  (`git ls-files --eol` → `i/crlf  w/lf`).
- В репозитории **нет** `.gitattributes`, `core.autocrlf` и `core.eol` не заданы,
  поэтому git не нормализует окончания строк — они зависят от редактора/ОС.
- Масштаб: в индексе **574 текстовых файла с CRLF** (Frontend3 — 375, из них
  188 `.js/.jsx`; Frontend2 — 47; backend — 68). Всего текстовых файлов ~1831.

## Scope (область)

- Новый файл `.gitattributes` в корне репозитория с политикой `eol=lf` для
  текстовых файлов и явной пометкой бинарных типов.
- Приведение к LF файла‑триггера `Frontend/Frontend3/src/utils/sellerCatalogI18n.js`
  (его рабочая копия уже LF — фиксируем в индексе, без изменения содержимого).
- (Опционально, по согласованию) единоразовая нормализация остальных CRLF‑файлов
  через `git add --renormalize .` — отдельным коммитом из‑за большого охвата.

## Не входит в задачу

- Любые изменения содержимого кода/логики.
- Переформатирование (Prettier/ESLint --fix) — только концы строк.
- Изменение поведения CI/сборки.

## Зависимости

- Нет.

## Риски

- Глобальная политика `text=auto eol=lf` ретроактивно влияет на отображение
  diff/`git status` для всех 574 CRLF‑файлов: они начнут показываться как
  изменённые до их фактической нормализации. Поэтому единоразовая нормализация
  выносится в отдельный коммит/решение по охвату.
- Файлы, которым CRLF нужен намеренно (`*.bat`, `*.cmd`), должны быть исключены.

- [x] `Frontend/Frontend3/src/utils/sellerCatalogI18n.js` приведён к LF и его
      `git diff --ignore-all-space` пуст (нет изменений содержимого).
- [x] Зафиксирован охват единоразовой нормализации (forward‑only / Frontend3 /

- [x] Подготовлено сообщение коммита, затрагивающего только файлы этой задачи.

# Iterations

## Iteration 1 — Analysis

### Цель
Понять причину и масштаб расхождения EOL.

### Действия
- `git diff --ignore-all-space` по файлу‑триггеру → подтвердить отсутствие
  изменений содержимого.
- `git ls-files --eol` → посчитать CRLF‑файлы по зонам.
- Проверить отсутствие `.gitattributes` и настроек `core.autocrlf`/`core.eol`.

### Output
- Подтверждение: расхождение только в EOL; 574 CRLF‑файла в индексе.

### Статус
- [x] Analysis complete

---

## Iteration 2 — .gitattributes

### Цель
Зафиксировать стандарт LF на будущее.

### Действия
- Создать `.gitattributes` в корне:
  - `* text=auto eol=lf`
  - бинарные типы помечены `binary`
  - `*.bat`, `*.cmd` → `eol=crlf`
- Привести файл‑триггер к LF и зафиксировать в индексе.

### Output
- `.gitattributes`, нормализованный `sellerCatalogI18n.js`.

### Статус
- [x] .gitattributes done

---

## Iteration 3 — Охват нормализации (решение)

### Цель
Выбрать стратегию для остальных 574 CRLF‑файлов, не раздувая текущий коммит.

### Варианты
- **Forward‑only (по умолчанию для этого коммита):** только `.gitattributes`,
  docs и файл‑триггер. Остальные файлы нормализуются лениво при их следующем
  редактировании. Текущий коммит остаётся узким.
- **Frontend3‑only:** `git add --renormalize Frontend/Frontend3` отдельным
  коммитом (~375 файлов).
- **Весь репозиторий:** `git add --renormalize .` отдельным коммитом (~574).

### Решение
- **Forward‑only.** Эмпирически проверено: добавление `.gitattributes` НЕ
  раздувает рабочее дерево — git не делает массовую ренормализацию без явного
  `git add --renormalize`. После создания `.gitattributes` как изменённый
  показывается только файл‑триггер (`sellerCatalogI18n.js`, у которого рабочая
  копия уже была LF). Остальные ~573 CRLF‑файла остаются нетронутыми и будут
  нормализованы лениво при их следующем редактировании.
- Полная единоразовая нормализация (`git add --renormalize .`, ~574 файла)
  вынесена в опциональный отдельный коммит на будущее и в этот коммит не входит.

### Статус
- [x] Decision recorded
