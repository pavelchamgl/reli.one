Listing Products

How to add a product to the catalog after your seller application is approved. Products are moderated before publication.

Version: June 2026

1. Prerequisites

  • Seller account created and email confirmed;
  • Onboarding completed, application status is Approved;
  • You can log in to the dashboard: https://reli.one/seller/login.
Before application approval, product management sections are unavailable or redirect to onboarding.

2. “Goods” section

After logging in, open the Goods section. The start page offers two actions:

  • List of goods — view products you have already created;
  • Adding goods — create a new product.

3. Creating a product

Click Adding goods or go to https://reli.one/seller/seller-create.

3.1. Required fields

Some fields are the same for all products. Others — category attributes — are loaded automatically after you select a category and may differ for different categories and subcategories.

Always required

FieldDescription
CategoryCatalog category. Usually select a leaf (final) category; in some branches, products are also allowed in a parent category.
Goods nameProduct name
DescriptionProduct description
PhotoAt least one image (JPG, PNG, WebP)
VAT rate, %VAT rate. If left empty, the system saves 0%.
Variant type nameVariant type name (e.g., “Size”, “Color”)
VariantsAt least one variant. For each variant, required: value, price, stock quantity, package weight, and dimensions (length, width, height) — for shipping calculation.

Category and subcategory attributes

After you select a category, the form loads an attribute schema. It includes:

  • attributes defined for the selected category;
  • attributes from parent categories (inherited in subcategories);
  • when an attribute code matches in a child category, the subcategory definition is used.

Only fields marked as required in the schema must be filled in (types: text, number, yes/no, select from list). If a category has no required attributes, you can skip this block. When you change the category, the schema reloads — review the fields again.

The Characteristics block (custom “name — value” pairs) is optional. However, if you start filling in a characteristic row, you must provide both name and value.

3.2. Optional fields

  • Brand — brand name (text, at least 2 characters);
  • Seller article — seller SKU (digits only, at least 10). If omitted, the system generates an article automatically;
  • EAN/UPC barcode — barcode;
  • Additional details — additional information;
  • Country of origin — country of origin;
  • Warranty, months — warranty in months (integer > 0);
  • 18+ / age restricted — age restriction flag;
  • License / Certificate — certificate or license (PDF, JPG, PNG, WebP, up to 13 MB).

4. Preview and submit for moderation

After completing the form, click Preview at the bottom of the product creation page — the preview screen opens. Review the description, characteristics, variants, and images, then click Sending for moderation.

The product status becomes On moderation. Typical moderation time is about 24 hours. After approval, the product appears in the public catalog with status Active.

5. Pre-submission checklist

  1. Category is correct; all required category and subcategory attributes are filled in.
  2. Description is clear, typo-free, and includes key characteristics.
  3. Photos are sharp, 1:1 aspect ratio, product is clearly visible.
  4. Each variant has value, price, stock, and package dimensions.
  5. VAT rate is correct; seller article is verified (or left for auto-generation).
  6. Certificate / license attached if required.

6. Common issues

ProblemSolution
Category attribute schema does not load Reselect the category or refresh the page. Make sure you selected a category where product creation is allowed
Expected attributes are missing Attributes depend on category: some are defined in the parent category, others in the subcategory. Select the exact product category
Submit button inactive on preview Check required attributes and variants
Product stuck “On moderation” Wait for moderator decision; if delayed > 48 h — contact support
Product rejected Fix the feedback and resubmit via Edit

7. Managing the product list

All your products are available at https://reli.one/seller/goods-list. Use tabs to filter:

  • Active — approved and published;
  • On moderation — awaiting review;
  • Not moderated — drafts and rejected items.

For each product, Edit (edit) and Del (delete) are available. When you edit an approved product, its status changes to Pending again — the product is temporarily removed from the catalog until re-moderation.
